# 计算机图形学课程实验五：光线追踪与镜面反射

项目简介：本项目基于 Python 和 Taichi 实现，完成了一个简单的 Whitted-Style 光线追踪渲染程序。

程序通过从相机向每个像素发射主光线，检测光线与场景中物体的交点，并进一步发射暗影射线和反射射线，实现了硬阴影和理想镜面反射效果。场景中包含黑白棋盘格地面、红色漫反射球和银色镜面球，可以通过 UI 滑块实时调节光源位置、最大反射次数和镜面反射强度。

运行方式：

```bash
uv run python src/Work05/main.py
```

## 二、代码逻辑

1. 初始化阶段

（1）使用 Taichi 初始化运行环境，并创建像素缓冲区 `pixels`。

（2）定义光源位置参数 `light_x`、`light_y`、`light_z`，用于控制光源在三维空间中的位置。

（3）定义最大反射次数 `max_bounces`，用于限制镜面反射射线的继续传播次数。

（4）定义材质类型常量 `MAT_DIFFUSE` 和 `MAT_MIRROR`，用于区分漫反射材质和镜面材质。

2. 场景求交

（1）程序从相机位置向每个像素发射一条主光线。

（2）使用 `intersect_sphere()` 判断光线是否与球体相交。

（3）使用 `intersect_plane()` 判断光线是否与水平地面相交。

（4）在 `scene_intersect()` 中统一检测场景中所有物体，并保留距离相机最近的交点。

（5）地面位于 `y = -1.0`，通过交点的 `x` 和 `z` 坐标计算黑白棋盘格纹理。

3. 阴影与反射

（1）当光线击中漫反射物体时，程序从交点向光源方向发射一条暗影射线。

（2）如果暗影射线在到达光源之前击中了其他物体，则说明该点处于阴影中，此时只保留较弱的环境光。

（3）如果光线击中镜面球体，则根据反射公式计算新的反射方向，并继续追踪反射射线。

（4）为了适合 GPU 并行执行，程序没有使用递归，而是使用固定次数的循环实现迭代式光线追踪。

4. 渲染与交互

（1）每一帧执行 `render()`，为所有像素并行计算颜色。

（2）使用 `canvas.set_image(pixels)` 将像素缓冲区绘制到窗口中。

（3）右上角 UI 面板可以实时调节光源位置、最大反射次数和镜面反射强度。

（4）按下 `R` 键可以恢复默认参数。

## 三、实现功能

（1）实现从相机出发的主光线发射。

（2）实现光线与球体的相交检测。

（3）实现光线与无限大地面的相交检测。

（4）实现黑白棋盘格地面纹理。

（5）实现红色漫反射球渲染。

（6）实现银色镜面球渲染。

（7）实现硬阴影检测。

（8）实现理想镜面反射。

（9）实现迭代式光线追踪，避免递归写法。

（10）实现光源位置、最大反射次数和镜面强度的 UI 调节。

| 操作 | 功能 |
| --- | --- |
| 拖动 `Light X` | 调节光源 x 坐标 |
| 拖动 `Light Y` | 调节光源 y 坐标 |
| 拖动 `Light Z` | 调节光源 z 坐标 |
| 拖动 `Max Bounces` | 调节最大反射次数 |
| 拖动 `Mirror` | 调节镜面反射强度 |
| `R` | 恢复默认参数 |
| `Esc` | 退出程序 |
